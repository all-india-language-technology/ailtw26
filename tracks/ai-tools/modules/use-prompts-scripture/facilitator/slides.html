Your Reference Card: Anatomy of a Good Prompt

  • R Role — Who is the AI acting as?
    "You are a biblical scholar specialising in…"
  • C Context — What does it need to know?
    "We are translating [Book] for a Tibeto-Burman language…"
  • T Task — What exactly are you asking?
    "List the 3–5 key interpretive questions for [verse]…"
  • C Constraints — Format, length, source?
    "Base your answer only on the uploaded document. Use plain language."

New Skill 1: Upload a Document

📎
Upload a commentary excerpt, lexicon entry, or project notes.
Then ask: "Based only on this document, explain…"
🔑 The AI reasons from your source — not its general training.
You control the frame of reference.

Watch for

  • 1How the response changes when you ground it in a specific source
  • 2What happens when you add "only from this document" as a constraint
  • 3How to verify the answer points back to the document
💡 Follow along on your own device if you can.
Open the AI tool live. Upload a prepared document (commentary excerpt works well). Ask a question grounded in that document. Show the output. Narrate: "Notice I added 'based only on this document' — without that, it ignores our source." Advance ↓ to Demo 2.
Content

New Skill 2: Paste a Link

🔗
Paste a URL as your reference source.
"Using this article, summarise the background to [passage]…"
⚠️ Not all tools read links the same way.
Some fetch the page; some don't. Test before the session.

When to use each

  • 📎Upload — private files, PDFs, project documents, notes
  • 🔗Link — public web articles, online lexicons, open commentaries
→ You'll choose your method in Round 2. Note which your tool handles reliably.
Paste a prepared URL live. Show the response — compare briefly with file upload result. Emphasise: link-reading behaviour varies by tool and page accessibility. Participants should note which method their preferred tool supports reliably. Advance → to C3 (Challenge).

Round 1: Building the Prompt

The prompt sequence

  • R Role: "You are an exegetical consultant helping a Bible translation team…"
  • C Context: "The team translates [Book] into a Tibeto-Burman language. They use GNTD as their primary source…"
  • T Task: "List 4–5 key exegetical questions the team should verify for [passage]…"
  • C Constraint: "Base your analysis on the uploaded document only. Use plain, non-technical language."
🔨 Building live — follow along on your device or watch. Ask questions after.
💭 Note one thing you'd change or add. You'll share it in the debrief.
Think aloud throughout. Narrate every decision: "I'm giving it a role because the team needs exegetical, not devotional, output." "I'm asking for options, not an answer — because the team needs to decide." "I'm uploading the GNTD notes so it reasons from their source, not a general gloss." Show the output. Let the room see and react before moving to debrief. Advance ↓ to debrief.
